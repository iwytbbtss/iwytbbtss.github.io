page2-자기소개
마이스터 고등학교에서 정밀기계를 전공하고 '[디지털 컨버전스] 프로그레시브 웹앱 프론트엔드 개발 실무자 양성과정'을 통해 개발자로서 공부를 시작했습니다.
새로운 기술을 배우고 적용해보는 것을 좋아합니다. Frontend 개발자로서 React와 Vue.js는 물론 카카오맵 API, Firebase, 웹 크롤링, TypeScript, SwiperJs 등 새로운 기술을 계속 배우려고 노력했고 실제 프로젝트에 적용시킨 경험이 있습니다.
동기생들과 팀 프로젝트에서 주로 팀장을 맡았습니다. 팀원들이 적극적으로 커뮤니케이션에 참여할 수 있도록 노력했고 역량에 맞는 역할 분배, 구성원들의 코드 리뷰를 통해 주어진 기간안에 두 개의 프로젝트를 완성한 경험이 있습니다. 
반복되는 일을 줄이고 코드의 가독성, 재사용성을 높이는 것을 좋아합니다. 자주 사용하는 컴포넌트들(사이드바, 차트, 모달창, 이미지카드 등)을 재사용 가능한 컴포넌트로 보유하고 있으며 개발 과정 중에도 보일러 플레이트 코드를 줄이기 위한 노력을 하고 있습니다.
디버깅과 성능을 개선하는 일에 관심이 많습니다. 어떠한 사용자의 입력에도 비정상적인 작동을 하지 않도록 예상 가능한 모든 예외처리를 하기 위해 노력하며 원활한 사용자 경험을 만들기 위해 무거운 라이브러리(carousel, fullpage, cheerio)를 버리고 필요한 기능을 직접 만들어 사용한 경험이 있으며, 지속적인 CS 공부와 알고리즘 공부를 병행하고 있습니다.

개선/문제해결 사례(문제, 원인, 연구, 해결, 평가, 비고)
1. 웹크롤링 모바일 성능 최적화 사례
문제 1) 모바일에서 크롤링한 데이터가 출력되지 않는 이슈
문제 2) 크롤링한 데이터의 출력이 2초 이상 걸려 사용자 경험이 저하되는 이슈
원인 1) 모바일에서 HTTP요청으로 받아오는 HTML이 PC와 달라 발생
원인 2) 기존 라이브러리(cheerio)의 성능이 느려 발생
해결 1) HTTP request headers의 User-Agent 속성을 Windows로 설정
해결 2) 단순 parsing만 해주는 대신 성능이 우월한 라이브러리(parse5) 채택, 필요한 메서드 직접 구현
평가) 모바일 출력 이슈 해결, 데이터 출력 시간이 1초 미만으로 줄어듬
2. CORS에러를 Proxy서버를 만들어 우회한 사례
문제) 네이버 영화를 크롤링 중 CORS 에러 발생 이슈
원인) 요청을 보내는 곳과 받는 곳의 Origin이 달라 발생
해결) 오픈소스인 cors-anywhere를 받아 Heroku를 통해 서비스, 요청 우회
평가) CORS 정책을 우회해 HTTP 요청 받기 성공
비고) 상업적으로 사용히 불가능해 실제 서비스 시 백엔드를 통해 요청을 보내야 함
3. Firebase CLI와 Github actions를 활용해 프로젝트의 CI/CD를 구현한 사례
4. Vue.js로 완성된 프로젝트 렌더링 성능 최적화 사례
